# anomaly_analyzer

**anomaly_analyzer** — это ETL-сервис на Python для высокопроизводительного анализа потоков данных из Apache Kafka. Он предназначен для обнаружения аномалий в больших объемах сообщений, агрегации статистики и отправки уведомлений о результатах в чат (например, Telegram), основываясь на проекциях, полученных от Apache Flink.

## 🚀 Основные возможности

- **Потоковая обработка**: Непрерывное потребление сообщений из топиков Kafka.
- **Выявление аномалий**: Анализ входящих данных на основе заданных правил и проекций Flink для обнаружения отклонений.
- **Агрегация статистики**: Сбор и вычисление ежедневной статистики по обнаруженным аномалиям.
- **Управление партициями**: Интеллектуальное управление чтением из партиций Kafka для обеспечения отказоустойчивости и масштабируемости.
- **Уведомления в чат**: Автоматическая отправка сводок и алертов о найденных аномалиях в мессенджер.
- **Модульная конфигурация**: Гибкая настройка через YAML-файл.

## 🏗 Архитектура и компоненты

Проект состоит из следующих ключевых модулей:

| Модуль | Назначение |
|--------|------------|
| **`main.py`** | Точка входа в приложение, оркестрирующая запуск всех компонентов |
| **`kafka_consumer.py`** | Отвечает за подключение к Kafka, подписку на топики и получение сообщений |
| **`message_converter.py`** | Преобразует сырые сообщения из Kafka во внутренний формат и содержит основную логику анализа и выявления аномалий |
| **`anomaly_analyzer.py`** | Инициализирует настройки и компоненты. Запускает основной цикл для выполнения нужных действий с обработанными аномалиями |
| **`daily_statistic.py`** | Собирает и агрегирует статистику за день |
| **`anomaly_notifaer.py`** | Формирует и отправляет уведомления в целевой чат |
| **`partition_manager.py`** | Управляет созданием потоков для обработки партиций и сообщений |
| **`partition_storage.py`** | Хранит вычитанные, но необработанные сообщения для восстановления после перезапуска |
| **`config_parser.py`** | Обеспечивает загрузку параметров конфигурации |
| **`config.yaml`** | Файл с настройками сервиса |

## ⚡ Быстрый старт

### Требования

- Python 3.12 или выше
- Доступ к работающему кластеру Apache Kafka
- (Опционально) Настроенный бот для отправки сообщений в чат

### Установка и запуск

1. **Клонируйте репозиторий**
2. **Установите зависимости**
3. **Настройте конфигурацию** config.yaml
4. **Запустите сервис**
